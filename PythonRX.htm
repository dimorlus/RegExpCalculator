Python RegExp

 Top  Next

 

Python RegExp engine emulates Python's re module syntax. In this implementation, it uses std::regex internally, so it supports similar features to Modern engine but with Python-style replacement syntax.

Search expression (Python re syntax):

Pattern

Description

.*? .+?

Non-greedy (lazy) quantifiers - match as few characters as possible.

(?P<name>...)

Named capture group (not supported in current implementation).

(?=...) (?!...)

Positive and negative lookahead (same as Modern).

\d \w \s

Character classes: digit, word character, whitespace (same as Modern).

\b

Word boundary.

{n,m}

Repetition quantifier.

 

 

Replace expression (Python re.sub syntax):

Pattern

Description

\0

Entire matched pattern (full match).

\1..\9

Captured groups 1-9. Maximum 9 groups supported.

\\

Literal backslash \.

 

Key difference from Modern:

Python uses \0, \1-\9 for replacements, while Modern uses $&, $0-$9.

 

Note:

This implementation converts Python replacement syntax to ECMAScript internally.
