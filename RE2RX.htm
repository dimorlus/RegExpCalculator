RE2 (Google) RegExp

 Top  Next

 

RE2 is Google's regular expression library designed for guaranteed linear time execution. It protects against ReDoS (Regular Expression Denial of Service) attacks by limiting features.

Search expression (RE2 syntax):

Pattern

Description

.*? .+?

Non-greedy (lazy) quantifiers supported.

\d \w \s

Character classes supported.

(?:...)

Non-capturing group.

{n,m}

Repetition quantifier.

 

 

NOT SUPPORTED (for linear time guarantee):

\1 \2

Backreferences (cannot be matched in linear time).

(?<=...) (?<!...)

Lookbehind assertions (not supported).

 

 

Replace expression (RE2 syntax):

Pattern

Description

\0

Entire matched pattern (full match).

\1..\9

Captured groups 1-9. Maximum 9 groups supported.

\\

Literal backslash \.

 

Key advantages:

- Guaranteed O(n) linear time complexity

- Protection against ReDoS attacks

- Fast and predictable performance

 

Key limitations:

• No backreferences in patterns (\1, \2, etc.)

• No lookbehind assertions

• Backreferences work only in replacement strings

 

When to use RE2:

Use RE2 when processing untrusted input or when predictable performance is critical. For complex patterns with backreferences, use Modern or Python engines.
