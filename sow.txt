Я хочу сделать новую версию своего старого приложения (CBuilder 6) под Windows - калькулятор регулярных выражений, 
шаблонов для printf и scanf RegExp evaluator. Я хочу использовать VC2022 и делать  Win API приложение. Цель - 
получить один исполняемый файл минимального размера (и один готовый файл помощи в формате .chm) без каких-либо 
зависимостей (никаких runtime библиотек и т. п.). 

Приложение представляет собой окно, с тремя или двумя строками ввода для ввода исходной строки, выражения поиска 
и выражения замены с историей прошлых выражений и полем под ним для вывода результатов замены, поиска, и найденных 
групп для режима regexp, и две строки ввода для шаблона (формата) и данных для режимов scanf и printf. Результат 
в этом случае обычно ограничивается одной строкой (хотя printf может и несколько вернуть). Режим RegExp, Printf, 
Scanf задается меню с radio button. Так же через меню задаются опции, например выбор того или иного regexp engine. 
Истории строк ввода для каждого режима должны быть свои и сохраняться вместе с выбором режима в реестре. Поведение 
строк ввода с историей стандартное для ComboBox, результат - read only EditText. В строках ввода и окне вывода 
должен использоваться моноширинный шрифт "Courier New" 14pt. Ширина окна приложения по умолчанию должна вмещать 80 
символов в EditText, однако может растягиваться или сжиматься пользователем. При этом длинные строки в поле вывода 
просто обрезаются (горизонтальный скроллер не нужен. Координаты и выбранная ширина (минимум на 40 символов, 
максимум - сколько позволяет экран) должна запоминаться в реестре при закрытии окна программы и восстанавливаться 
из него при следующем запуске. Приложение должно корректно работать на экранах с разным dpi и с разными размерами 
шрифта. Горячими клавишами Ctrl++/- размер шрифта можно менять. Горячими клавишами Ctrl+Shift++/- можно менять 
прозрачность окна. Выбранный пункт меню Always on top должен закреплять окно программы поверх остальных окон. 
Клавиши Ctrl+Home должно помещать окно в координаты 100:100, устанавливать 14pt шрифт, ширину в 80 символов и делать 
окно полностью непрозрачным - то есть default value. И строки ввода, и поле вывода должны иметь белый цвет, черный 
шрифт, минимально возможного размера рамки и отступы от границ окна, меню и друг друга. Строки ввода должны иметь 
подписи. 
 Для regexp:
	Source:
	 [.................]
	RegExp:
	 [.................]
	Replace:
	 [.................]
 Для printf:
	Format:
	 [.................]
	Parameters:
	 [.................]
 Для scanf:
	Format:
	 [.................]
	Data:
	 [.................]

Окно истории должно выпадать вниз от строки, к которой оно относится, занимать всю ее ширину, вмещать 5 строк истории (если они есть),
показывать вертикальный скроллер, если строк больше (всего надо сохранять до 100 строк, дальше - вытеснять). Горизонатльный скроллер 
не нужен, если строка длинее, чем помещается в окно истории, пказывается только та ее часть, что помещается в него. Клавища Del в 
окне истории должна удалять текущую строку из истории.

 От похожего по интерфейсу и идеологии калькулятора математических формул. Данное приложение - в пару к нему, должно быть 
 максимально на него похожим, и в будущем будет устанавливаться вместе с ним.
 
В зависимости от регулярного выражения, строки, к которой оно применяется, и используемого regexp engine в поле результата
может быть разное число строк, общее число, я думаю, до 20. Для первого regexp engine они должны выглядеть как
Res:
  &:
 \0:
 \1:
 ...
 \A:
 ...
 \F:
Строка Res: должна быть видна всегда, даже если она пустая, остальные - только если они не пустые. 

Любое изменение любой из строк ввода должно немедленно "пересчитываться" и вызывать изменение результата. Все текстовые 
контролы должны поддерживать стандартные Copy/Paste операции (поле вывода только Copy, так как оно R/O).
