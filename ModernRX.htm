Modern RegExp (ECMAScript)

 Top  Next

 

Modern RegExp engine uses C++ std::regex with ECMAScript syntax (JavaScript-compatible). Supports lazy quantifiers and lookahead/lookbehind assertions.

Search expression (ECMAScript standard):

Pattern

Description

.+?

Lazy quantifier - matches as few characters as possible. For example, <.+?> in <a><b> matches <a> (not <a><b>).

*? ?? {n,m}?

All quantifiers can be made lazy by adding ? after them.

(?=...)

Positive lookahead - matches if pattern inside is ahead. For example, q(?=u) matches q only if followed by u.

(?!...)

Negative lookahead - matches if pattern inside is NOT ahead.

\d \w \s

Character classes: \d = digit [0-9], \w = word character [A-Za-z0-9_], \s = whitespace.

\D \W \S

Negated character classes (uppercase = opposite of lowercase).

\b \B

Word boundaries: \b = word boundary, \B = not word boundary.

{n,m}

Matches between n and m repetitions. For example, a{2,4} matches aa, aaa, aaaa.

 

 

Replace expression (ECMAScript standard):

Pattern

Description

$&

Entire matched pattern (full match).

$0..$9

Captured groups (0 = full match, 1-9 = groups). Maximum 9 groups supported.

$$

Literal dollar sign $.

$`

Text before the match.

$'

Text after the match.

 

Common patterns supported by all engines (see Classic for full list):

^ $ . * + ? [ ] ( ) | \
